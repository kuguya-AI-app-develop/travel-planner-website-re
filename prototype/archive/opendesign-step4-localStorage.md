# Travel Planner 多计划改造 - 第4步：localStorage 持久化

## 上下文
在前三步的基础上，已经完成了多计划数据结构、计划选择器、计划管理、计划总览和对比功能。

## 本次任务
添加 localStorage 持久化，确保数据刷新不丢失。

## 具体要求

1. 存储：
   - 使用统一 key 'travel-planner-data' 存储整个 plans 对象和 activePlanId
   - 封面图片使用独立 key 'tp-cover-{planId}' 存储，避免主数据过大
   - 所有数据修改操作（增删改、勾选、评分、拖拽、切换计划、编辑字段）自动触发保存
   - 使用全局 blur/change 事件监听兜底

2. 加载：
   - 页面初始化时从 localStorage 恢复
   - 恢复成功显示轻提示「已从本地恢复数据」
   - 无存储数据时使用默认示例数据

3. 重置：
   - 侧边栏底部添加「重置数据」按钮（红色文字）
   - 点击需确认，确认后清除所有 localStorage 并刷新

4. 兼容性：
   - 如果检测到旧版单计划格式数据，自动迁移为多计划结构（包裹为一个默认计划）

## 请输出
完整的 HTML+CSS+JS 文件，包含前三步的所有功能 + 本次新增功能。
