# Travel Planner 多计划改造 - 第1步：数据结构 + 计划选择器

## 上下文
我有一个旅行策划单页 HTML 原型，目前所有数据（行程、机票、目的地、酒店、消费、待办）都是全局变量，只支持一个计划。

## 本次任务
将数据模型改造为支持多计划，并在侧边栏顶部添加计划切换器。

## 具体要求

1. 数据结构改造：
   - 将所有业务数据包裹到 plans 对象中，每个 plan 有独立的 trips、flights、flightCriteria、destinations、destCriteria、hotels、hotelCriteria、expenses、checklistItems、nextId
   - 全局维护 activePlanId 记录当前选中的计划
   - 提供 switchPlan(planId) 函数，切换时刷新所有模块

2. 侧边栏计划选择器：
   - 放在侧边栏品牌名称下方
   - 显示当前计划名称，点击展开下拉列表
   - 列表中每个计划显示名称和状态标签
   - 点击切换计划，下方所有模块数据自动刷新

3. 示例数据：
   - 预置 2 个计划：「东京购物之旅」（有行程、航班、酒店等完整数据）和「京都周末行」（只有部分行程和目的地数据）
   - 两个计划的数据要明显不同，方便验证切换效果

4. 保持现有设计风格：oklch 色彩、Inter 字体、圆角卡片、左侧固定侧边栏、响应式布局

## 请输出
完整的 HTML+CSS+JS 文件，可直接在浏览器打开运行。其余功能模块（行程日历、机票对比、目的地、酒店、消费、预算总结）保持原有布局不变。
